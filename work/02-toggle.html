01 — Context

왜 이 프로젝트를 했는가

Toggle은 Gmail, Slack, Notion 등 분산된 업무 툴을 하나의 플랫폼에서 통합 관리할 수 있는 B2B SaaS 서비스입니다. 시드 투자 유치 후 서비스 성장 단계에서 핵심 지표인 홈 화면 체류시간이 평균 3초 미만으로 낮게 나타나는 문제가 있었습니다.

외부 디자이너 8인 스프린트 구조로 진행된 프로젝트로, 개인별 독립 솔루션 제출 후 상호 피드백하는 방식으로 운영되었습니다.

02 — Problem

문제의 크기

기존 홈 화면은 통합 검색 기능만 제공하고 있어 사용자가 업무 흐름을 파악하거나 원하는 정보를 찾기 위해 반복적으로 화면을 이탈하는 구조였습니다.

홈 화면에서 사용자의 행동이 검색에만 제한되면, 통합 솔루션에 대한 필요성을 느끼지 못해 재접속률까지 낮아지고 서비스 리텐션이 떨어집니다. 단순 체류시간 문제가 아니라 서비스의 핵심 가치 전달 실패였습니다.

  • 낮은 체류시간 — 홈 화면 평균 체류시간 3초 미만, 통합 검색 외 활용 가능한 인터페이스 부재
  • 주요 기능 화면 이탈 — 업무 흐름 파악을 위해 각 기능 페이지로 이동해야 하는 구조, 홈 화면이 허브 역할을 못함
  • 서비스 가치 미전달 — 연동된 비즈니스 서비스를 활용하기 어려운 구성으로 '효율적인 업무 관리'라는 가치를 체감할 수 없음
03 — Decision

무엇을 하고, 무엇을 하지 않았는가

핵심 판단은 통합 검색을 제거하지 않되, 홈 화면의 역할을 '검색 진입점'에서 '업무 허브'로 재정의하는 것이었습니다. 체류시간 6초 이상을 Target Metric으로 설정했습니다.

비즈니스 목표인 서비스 리텐션 개선을 솔루션 설계의 기준으로 연결했습니다. 단순히 "화면을 예쁘게" 만드는 것이 아닌, 사용자가 홈 화면에서 오래 머물 이유를 만드는 것이 핵심이었습니다.

  • 한 것 — 프로젝트 단위 업무 데이터 분류와 칸반보드 기반 홈 화면으로 재설계. 연동된 SaaS 데이터가 즉시 보이는 구조로 전환
  • 하지 않은 것 — 각 기능 페이지의 완전한 재설계. 홈 화면이 모든 기능을 보여주려는 욕심을 버리고 '업무 흐름 파악'에만 집중
  • 기준 — 사용자의 익숙한 칸반 인터랙션 패턴을 그대로 활용, 학습 비용 최소화

익숙한 인터랙션 패턴 활용
